## Work Done

- Wired `recordTokenUsage()` into StreamingPipeline.processEvent() after user handler dispatch
- Wired session pool metrics into SquadClient:
  - `recordSessionCreated()` after successful `client.createSession()` return
  - `recordSessionClosed()` after successful `client.deleteSession()` return
  - `recordSessionError()` at innermost catch blocks
- Metrics recorded for EVERY failed attempt, including ones that trigger reconnection
- No new exports needed — barrel and subpath exports already wired in Phase 1

## Key Decisions

1. **Token metrics after user handlers:** User handlers see events before OTel instrumentation; handler failures don't block metric recording
2. **Session metrics at success/error boundary:** Innermost boundary ensures accurate failure counts and prevents double-counting
3. **Error recording includes reconnect attempts:** Reconnect-eligible failures are still errors worth counting
4. **Zero overhead for non-OTel users:** Metric calls are no-ops when OTel not configured
