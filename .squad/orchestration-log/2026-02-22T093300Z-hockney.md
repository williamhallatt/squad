- Tests verify exact metric names, values, and attributes without real OTel SDK or collector

## Key Decisions

1. **Spy meter pattern avoids async complexity:** No need for InMemoryMetricExporter with complex flush semantics
2. **Synchronous and fast:** Keeps tests efficient and deterministic
3. **Pattern consistent with otel-bridge:** Spy spans via InMemorySpanExporter, adapted for metrics API surface
4. **Pattern for future OTel metric tests:** All future metrics tests should follow this same approach
