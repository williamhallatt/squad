# Rock Paper Scissors — Multi-Agent Strategy Arena

⚠️ **Experimental alpha.** This sample demonstrates beta SDK features with live Copilot sessions.

Multiple LLM-backed agents with distinct strategies compete in endless 1-on-1 matches. A scorekeeper agent provides real-time commentary. All responses generated by live Copilot LLM sessions — no demo mode, no canned output.

## What It Shows

- **Multi-agent session management** — 8+ concurrent Copilot LLM sessions (one per player, one for scorekeeper)
- **Strategy via prompt** — Each agent's personality and strategy defined entirely by its system prompt
- **Agent learning** — Sherlock analyzes opponent play history to detect patterns and predict moves
- **Real-time streaming** — Scorekeeper commentary streams token-by-token via `StreamingPipeline`
- **Tournament structure** — Random pairings cycle through all possible matchups; history tracked per pair for learning
- **SDK patterns shown:**
  - `SquadClientWithPool`: auth and session pooling
  - `CastingEngine`: casting a roster of agents
  - `StreamingPipeline`: token-streaming output
  - `SessionPool`: managing multiple concurrent sessions
  - `EventBus`: subscribing to session lifecycle events

Perfect for:
- Learning real-world Squad SDK usage
- Understanding LLM-powered multi-agent systems
- Seeing strategy and learning in action
- Exploring streaming output in realistic scenarios

## How It Works

### Architecture

1. **Client & Auth** — `SquadClientWithPool` establishes authenticated connection to GitHub Copilot
2. **Session Pool** — Creates 9+ concurrent Copilot LLM sessions (one per player, one scorekeeper)
3. **Match Loop** — Tournament engine:
   - Randomly pairs two players
   - Sends each player their system prompt + opponent's history
   - LLM generates move (or reasoning + move for Sherlock)
   - Compares moves, determines winner
   - Tracks score for this pairing
4. **Scoring** — Per-pairing history (e.g., Sherlock vs. Rocky maintains its own win count)
5. **Scorekeeper** — Receives move results, generates entertaining commentary via streaming
6. **EventBus** — Session creation, message sending, and response streaming subscribed for monitoring

### Key SDK Concepts

- **`SquadClientWithPool`** — Manages authentication, session pooling, and resource limits
- **`CastingEngine`** — Cast a roster of agents with typed definitions (not used here; player strategies are hardcoded)
- **`SessionPool`** — Reuses sessions across multiple messages (one player session handles many matches)
- **`StreamingPipeline`** — Consumes `message_delta` events and renders token-by-token output
- **`EventBus`** — `subscribe()` to `session:created`, `message:streaming`, etc. for real-time monitoring

## The Learning Agent — Sherlock

Sherlock is the only agent with access to opponent history. Its system prompt instructs it to:

1. **Analyze opponent patterns:**
   - Frequency bias (e.g., "throws rock 70% of the time")
   - Sequences (e.g., "alternates rock-paper-rock-paper")
   - Trends (e.g., "favors scissors after losing")
   - Cycles (e.g., "repeats rock-paper-scissors-rock-paper-scissors")

2. **Predict next move** based on the analysis

3. **Counter strategically:**
   - If opponent will throw rock → Sherlock throws paper
   - If opponent will throw paper → Sherlock throws scissors
   - If opponent will throw scissors → Sherlock throws rock

4. **Output format:**
   ```
   [One sentence analysis]
   [Move: rock, paper, or scissors]
   ```

**Example reasoning in logs:**

```
Sherlock analyzing Rocky...
History: rock, rock, rock, rock, rock, rock (100% rock bias)
Sherlock responds: "Opponent throws rock every round. Paper wins.
paper"
```

Early rounds, Sherlock has limited history and may perform like Chaos. But after 10+ rounds against the same opponent, it often identifies patterns and achieves higher win rates against predictable players. Against Chaos (pure random), Sherlock converges to 33% win rate.

## Customizing

### Add a New Player

Edit `prompts.ts` and add a new entry to `PLAYERS`:

```typescript
{
  id: 'my-strategy',
  name: 'Agent Name',
  emoji: '🎯',
  strategy: 'Brief description of the strategy',
  systemPrompt: `You are playing rock-paper-scissors. [Your custom instructions].
  
  When asked for your move, respond with exactly one word: rock, paper, or scissors (lowercase).`,
}
```

Then restart the sample. The new player will be added to the roster and compete in the tournament.

### Adjust Match Settings

In `index.ts` (when implemented), you may configure:
- **Round limit per pairing** — How many rounds per matchup before moving to the next pair
- **Total matches** — Stop after N total rounds (or run forever)
- **Pairing strategy** — Random, round-robin, league, or custom
- **History window** — How many previous rounds Sherlock sees

### Modify Scorekeeper Behavior

The scorekeeper's system prompt is in `SCOREKEEPER_PROMPT` in `prompts.ts`. Customize tone, leaderboard format, or commentary style.

## Running the Tournament

The sample runs forever, continuously cycling through player pairings. Each match is 10 rounds (configurable). After each round, the scorekeeper announces the result. Every 10 rounds, the leaderboard updates.

To stop, press `Ctrl+C` in the terminal.

Monitor logs to see:
- Player moves and reasoning (especially Sherlock's analysis)
- Scorekeeper commentary
- Win streaks and upsets
- Pattern detection over time

## Troubleshooting

**`GITHUB_TOKEN not found`** — Set your token: `export GITHUB_TOKEN=ghp_...`

**`Authentication failed`** — Verify your token has Copilot access. Token must be from an account with an active Copilot subscription.

**`Session creation timeout`** — Network latency or rate limiting. Verify internet connection. Wait a moment and retry.

**`Moves not being generated`** — Check logs for LLM response errors. Confirm your Copilot token is valid and has not expired.

**`Scorekeeper silent`** — The scorekeeper session may be experiencing a delay. The LLM is reasoning about the match; wait a moment for commentary.
